# 贡献指南

感谢你对 mall-saas 的兴趣！本文档说明如何向本仓库提交贡献。

## 行为准则

参与本项目即代表你同意遵守 [Code of Conduct](./CODE_OF_CONDUCT.md)。请在所有交流中保持尊重与建设性。

## 前置依赖

- Node.js ≥ 22
- pnpm ≥ 9（`corepack enable && corepack prepare pnpm@9.15.0 --activate`）
- Docker（用于本地起 postgres + redis）

## 本地开发流程

```bash
git clone https://github.com/can4hou6joeng4/mall-saas.git
cd mall-saas
docker compose up -d            # postgres + redis
cp .env.example .env
pnpm install
pnpm --filter @mall/api exec prisma migrate deploy
pnpm dev                        # 并行启动后端 + 三前端（开热重载）
```

更多脚本与端口见 [README.md](./README.md)。

## 分支与提交规范

### 分支策略

- `main`：永远可发布的稳定分支
- `feat/m{N}-<topic>`：里程碑式特性分支（参考既有的 `feat/m32-playwright-e2e` 等）
- `fix/<topic>`：bug 修复分支

请基于最新 `main` 切分支，开发完成后通过 **PR** 合入。

### Commit message 规范

格式必须是 `type: 中文描述`，无 scope、无 issue 编号、无尾部追踪 ID：

```text
feat: 添加用户登录流程
fix: 修复订单金额计算错误
docs: 更新部署说明
test: 补全 storefront 优惠券单测
ci: 调整 docker build target
```

允许的 `type`：`feat / fix / docs / style / refactor / test / chore / build / ci / perf / revert`

**禁止**：
- 带 scope（`feat(auth):`）
- 带 issue/PR/工单编号（`fix: ... #123`）
- 任何 AI 标识（`Co-Authored-By: Claude`、`Generated by ChatGPT` 等）
- 多句尾随元信息

## 提交 PR 前的自检清单

```bash
pnpm typecheck   # TypeScript 严格模式
pnpm lint        # ESLint --max-warnings 0
pnpm test        # 全工作区单测 + 后端 e2e
pnpm build       # 全 workspace 构建
```

涉及后端/前端联动的变更，跑对应的里程碑验收脚本：

```bash
SKIP_PIPELINE=1 bash scripts/m{N}-acceptance.sh
```

PR 描述请说明：
- 解决什么问题 / 引入什么能力
- 涉及的层（后端 / 前端 / CI / 文档）
- 是否需要数据库迁移
- 是否影响公开 API（OpenAPI 是否需重新生成）

## 引入新里程碑

本项目按"里程碑链"演进。新里程碑（M{N}）的标准节奏：

1. 创建 `feat/m{N}-<topic>` 分支
2. 后端改动 + e2e 测试覆盖
3. 前端改动 + jsdom 单测
4. 写 `scripts/m{N}-acceptance.sh` 端到端验收脚本，须 shellcheck 通过
5. PR review 合入 main → 打 tag `vX.Y.0-m{N}`
6. Release notes 跟随 tag
